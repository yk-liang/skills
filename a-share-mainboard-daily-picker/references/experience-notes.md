# 用户经验沉淀区

存储 **playbooks.md 未覆盖的增量规则**。每次运行 Step 1 读取本文件。

规则与实时数据冲突时，**优先用实时数据**并在报告说明。过去经验仅作风险提醒，不能作为买入理由。

## A 股交易基本约束

- **最小交易单位 = 100 股（1 手）**：所有买卖建议的股数 = `floor(金额/股价/100)×100`，不足 100 股写"不足 1 手，不操作"
- **新建仓保护期**：持仓 ≤ 2 个交易日不轻动，除非极端条件（跌停/ST/重大利空/板块跌停潮）

## 数据 vs 解读纪律

**事实数据 >> 平台解读**。`reason_type` / 诊股 / AI 推荐 / 研报 都是二手解读：
- **可用于发现同类标的、列标签**（"同花顺标注：CPO+多芯光纤"）
- **不可作入选论证主体**（入选因果链必须由量价/板块共振/公告/财务/龙虎榜支撑）
- 区别："用它找什么"OK vs "用它说明为什么涨"不 OK

详细一手数据清单见 `playbooks.md`。

## 历史复盘衍生规则（来自 case-library.md）

| 规则 | 触发条件 | 动作 |
|---|---|---|
| **不做缩量启动后放量介入** | 近 5 日有连续涨停 + 5 日均量 < 20 日均量 | 回避池 |
| **不追连扳** | ≥ 3 涨停 + 自己没在底部上车 + 大盘转弱 | 回避池（软化版见 case-library） |
| **不冲动追开盘拉升** | 计划低吸但当日开盘已拉升 | 写"等回踩 X 日线" |
| **一字板别限价卖** | 一字涨停 + 竞价封板 < 50% | "挂跌停价果断出" |
| **大涨先减半** | 持仓亏损中但当日 ≥ 5% | "减一半，回落再补" |
| **诱多识别** | 尾盘高位放量回落 | 标"诱多疑似" |
| **不破不动** | 持仓还在 5/10/20 日线上 | "盘尾再决定" |
| **横盘不跌 = 强** | 大盘下跌 + 个股横盘 + 缩量 | 进"逆市不跌"观察池 |
| **缩量反包 = 抛压衰竭** | 上升趋势 + 昨日大阴放量 + 今日缩量阳反包 | 加分项 |

## 经验提取流程

当用户说"以后这种票别做了""今天这个判断错了"等：
1. 识别**可重用**经验（vs 一次性情绪）
2. 模糊语言 → 可观察条件（"涨太多" → 5D/20D/60D 阈值）
3. 归类（exclusion / scoring / checklist）
4. 追加到本文件，必要时同步 SKILL.md / scoring.md / playbooks.md
