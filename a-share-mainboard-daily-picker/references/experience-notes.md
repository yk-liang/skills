# User Experience Notes / 用户经验沉淀区

这个文件存储用户**已确认**的可执行规则。优先级高于通用规则，但**永远不能替代实时数据**。

## 使用方式
- 每次运行前，**在 SKILL.md Step 1 已经读取过本文件**。
- 把每条 note 归类成三种之一：
  1. **Hard exclusion**（硬过滤，直接剔除）
  2. **Score adjustment**（评分加减分）
  3. **Review checklist**（必须 explicit 检查的项）
- 如果某条 note 与官方实时数据冲突，**优先用官方数据**并在报告里说明冲突。
- 一条 note 仅是"过去经验/教训"时，作为风险提醒使用，而不能作为买入理由。

## 用户已确认的核心原则（默认遵守）

### 大方向
1. **主板优先**：默认只看 A 股主板，排除科创板、创业板、北交所、B 股。
2. **数据源优先级**：公告 > 同花顺/东财 > 券商研报数据；不用股吧/小作文/群截图/未署名传闻。
3. **账户背景**：13.5 万账户、有 1.3 万回撤背景；首要任务是控回撤+回撤修复，不博暴利。
4. **不追高、不重仓、不 all-in、留预备队**。
5. **可验证逻辑优先于题材情绪**：财报改善、现金流改善、订单/回购/业绩预告/政策直接受益 > 题材炒作。

### 仓位结构（用户认可的"分仓"体系）
- 中线对冲仓 40–50%（"账户稳定器"）
- 短线机动仓 ≤ 30%
- 现金预备队 ≥ 20%
- 大盘 C 级时短线仓位归零、现金 ≥ 70%
- "**短线越差，越不能动中线票**"（不要为了追短线把对冲卖了）

### 票型纪律
- 退潮期三不做：不做首板、不追反包、不做新题材
- 退潮期仓位公式：涨停家数 × 1% = 当日最大短线仓位
- 中线票 → 主仓必须满足 5 选 4 升级条件（指数、黄白线、量、赚钱效应、板块持续性）
- 任何短线票，止损 ≤ 3%
- 连续 2 笔亏损 → 当天停手
- 不打首板，除非 A 级盘且是龙头

### 卖出纪律（三段式）
- 强一致日卖 30% / 第一次放量震荡卖 30% / 跌破 5 日线全清
- 「不赚最后一个铜板，赚最容易赚的钱」
- "卖飞是成本，不是错误"

### 做 T 纪律
- 适合做 T 的票：小盘（≤ 100 亿）、活跃换手（3–5%）、上升通道
- **绝不做倒 T**（先卖后买）— 电网设备 ETF 卖飞案例
- 被套优先做正 T 解套

### 节假日纪律
- 春节节前一周清仓空仓、节前 2–3 天加仓、节后第一天再看

### 行为纪律
- 不听别人推荐
- 盘中不临时改变交易计划（本来低吸临时去追高 → 最致命亏损来源）
- 大盘指数突破时不要兴奋手痒频繁开仓
- 一定要等持仓赚到钱才可以开新仓

### 数据 vs 解读纪律（2026-06-04 沉淀）

**核心原则**：**事实数据 >> 平台解读**。任何"事后归因/编辑解读"字段都只能作为**参考**，不能作为判断依据。

- **同花顺 / 东方财富的 `reason_type`（涨停原因）只是站方编辑事后凑的标签**：
  - 涨停发生后，编辑从近期消息面里找一个"最像的"理由贴上去
  - 真正驱动涨停的可能是机构席位、游资抱团、技术突破、板块联动——只有内部人才知道
  - 多原因混杂如"多芯光纤+CPO+一季报增长"，到底哪个是主因？编辑也不知道
  - 冷门票编辑甚至会"硬编"原因（找不到就写"行业回暖+业绩预期"）

- **判断个股的真实驱动力，应该回到数据本身**：
  1. **量价**：开盘形态、封板时间、封板资金、换手率、今日开板次数
  2. **板块共振**：是否多只同类型票同方向走强（共振 = 真热点，单票拉升 = 资金游戏）
  3. **K 线结构**：是否处于均线启动 / 反包 / 突破 / 横盘启动等可识别模式
  4. **公告硬事实**：巨潮原文里的中标合同、回购、业绩预告、资产注入（这些有金额、占比、日期，可量化）
  5. **财务支撑**：扣非净利、经营现金流、毛利率——业绩支撑的板块才能走远
  6. **龙虎榜席位**：机构席位 vs 游资席位的偏好（这是真行为数据）
  7. **北向资金行业偏好**：北向是聪明钱，方向有参考价值

- **agent 报告时如何使用 `reason_type`**：
  - 可以**列出**："同花顺标注原因：CPO+多芯光纤+一季报增长"
  - **绝不能**写"该股属于 CPO 主线"或"业绩预期驱动"这种把站方解读当事实的措辞
  - 推论必须来自**自己看到的数据**："板块共振显示光纤光缆主线（亨通/长飞/中天 3 只主板同向）+ 业绩支撑（亨通 26Q1 扣非 +108%）"

- **更广义**：任何"诊股 / AI 推荐 / 题材标签 / 研报观点 / 投顾文章"都属于二手解读类信息，与 i问财同源——agent 可以**参考**，但**不能让它影响最终结论**。结论必须来自一手数据 + 用户已确认的规则。

## 历史复盘衍生规则（来自 case-library.md）
（详见 `case-library.md`，这里只列规则名，便于 agent 在选股时引用）

| 规则 | 触发条件 | 动作 |
|---|---|---|
| **不做缩量启动后放量介入** | 近 5 日有连续涨停 + 5 日均量 < 20 日均量 | 直接进回避池 |
| **不追连扳** | 近 5 日 ≥ 3 个涨停 + 最新一日仍涨停 | 直接进回避池 |
| **不冲动追开盘拉升** | 计划是回踩低吸但当日开盘已拉升 | 报告写"等回踩 X 日线"，不准建议追入 |
| **一字板别限价卖** | 一字涨停 + 集合竞价封板 < 50% | 建议"挂跌停价果断出" |
| **大涨先减半** | 持仓亏损中但当日大涨 ≥ 5% | 建议"减一半，回落再补" |
| **诱多识别** | 尾盘高位放量回落 | 报告写"诱多疑似" |
| **不破不动** | 持仓还在 5/10/20 日线上 | 默认建议"盘尾再决定" |
| **横盘不跌 = 强** | 大盘下跌 + 个股横盘 + 缩量 | 加入"逆市不跌"观察池 |
| **缩量反包 = 抛压衰竭** | 上升趋势 + 昨日大阴线放量 + 今日缩量阳反包 | 加分项 |

## 新经验导入区（agent 自动追加）

当用户提供新经验时（口语 → 规则），按以下格式追加到此处：

### Note N: [短标题]
- **原文/来源**：粘贴或概括用户原始经验
- **规则类型**：Hard exclusion / Score adjustment / Review checklist
- **适用场景**：每日选股 / 持仓修复 / 题材筛选 / 公告核验 / 追高回避
- **执行规则**：明确可执行判断（用观察条件而非情绪词）
- **失效条件**：什么情况下这条经验不适用
- **置信度**：高（多次验证）/ 中（1-2 次）/ 低（单次）

## 经验提取流程（用户口语 → 规则）

当用户说"以后这种票别做了""今天这个判断错了"等：

1. 识别**可重用**的经验（vs 一次性情绪）
2. 移除不可验证的部分（除非有公告/数据支撑）
3. 把模糊语言转成可观察条件：
   - "涨太多" → 5D/20D/60D 涨幅阈值
   - "逻辑不硬" → 是否有公告、财报、订单、政策、业绩验证
   - "风险大" → 波动率、换手率、减持、问询函、业绩下滑、现金流恶化
4. 归类（exclusion / scoring / checklist）
5. 追加到本文件，必要时同步更新 SKILL.md / scoring.md / playbooks.md
