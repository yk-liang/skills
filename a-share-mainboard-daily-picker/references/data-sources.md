# 数据源指南（agent-browser 实战）

## 总则：只用原始可信数据

**不允许使用任何"AI 中介"型数据源**（如同花顺 i问财、各类 AI 选股助手、ChatGPT 插件、券商 AI 助手）。这类工具会用大模型对查询做语义解析、对结果做摘要重写，**真值会被改写**。

只允许：
1. 直接读取**原始结构化页面**（DOM / accessibility tree 抽文本）
2. 必要时**截图 + vision OCR**（用户明确不归档截图，读完即弃；保留方式仅为对话内嵌）
3. 数据冲突时回到**最权威源**：公告 → 交易所原文；行情 → 多源交叉

每条数据都必须附 **来源 URL + 抓取时间**。宁可写"未核验"，也不要靠记忆或推测编造。

## 第一档：监管 / 公告原始源

### 巨潮资讯网 — http://www.cninfo.com.cn/
- 用途：A 股所有上市公司的公告、定期报告、业绩预告、问询函回复、减持/回购/异动公告
- 推荐入口：`http://www.cninfo.com.cn/new/disclosure/stock?stockCode=600000`（替换代码即可，深市同样支持），可看个股近期所有公告
- 抓取方式：accessibility tree 拿"公告标题 + 日期"列表；点开 PDF 标题后通常是 PDF.js 渲染——需要 vision 看截图，不要去硬抓 PDF 字节流
- 关注窗口：最近 30 个日历日

### 上交所 — http://www.sse.com.cn/disclosure/
### 深交所 — http://www.szse.cn/disclosure/listed/notice/
- fallback：仅在巨潮加载失败时使用

### 公告危险信号（命中 → 直接进回避池或要求二次核验）
- "股东减持""集中竞价/大宗交易减持""减持计划"
- "被立案调查""问询函""监管函""警示函""非标审计"
- "股票交易异常波动""无应披露而未披露事项"
- "诉讼""仲裁""担保逾期""债务违约"
- "股权质押 + 平仓预警""高比例质押"
- "业绩预告 - 预亏 / 大幅下滑"

### 公告利好信号（命中 → 加分，但**仍需核验金额、占比、可持续性**）
- "中标 / 重大合同"（金额占年营收 < 5% 不算硬催化）
- "回购方案""注销式回购"（> 1% 总股本才有意义）
- "业绩预告 - 大幅增长 / 扭亏"
- "员工持股计划""股权激励"（看行权价 vs 现价）
- "重大资产重组""资产注入"（注意停牌时长）

## 第二档：行情 / 财务 / 资金（原始页面）

### 东方财富 — https://quote.eastmoney.com/

**所有需要的 URL 模板**（agent 直接 navigate 进去抽原始 DOM）：

| 数据 | URL 模板 | 抓什么 |
|---|---|---|
| 个股行情 | `https://quote.eastmoney.com/<前缀><代码>.html`<br>前缀：sh / sz | 现价、涨跌、5/10/20/60 日涨幅、成交量、换手率、流通/总市值 |
| 个股 F10 | `http://emweb.securities.eastmoney.com/PC_HSF10/CompanySurvey/Index?code=SH600519` | 最新季报/年报：营收、归母、扣非、毛利率、经营现金流 |
| 个股资金流 | `http://data.eastmoney.com/zjlx/<代码>.html` | 主力净流入、北向持股、近 5 日资金方向 |
| 上证分时 | `https://quote.eastmoney.com/zs000001.html` | 黄白线相对位置（白=加权，黄=平均股价）+ 成交量 |
| 深证分时 | `https://quote.eastmoney.com/zs399001.html` | 同上 |
| 涨停板池 | `https://quote.eastmoney.com/center/gridlist.html#zt_pool` | 涨停股、连板数、封板金额、涨停原因、首次封板时间 |
| 跌停板池 | `https://quote.eastmoney.com/center/gridlist.html#dt_pool` | 跌停股、跌停原因 |
| **概念板块排行** | `http://quote.eastmoney.com/center/boardlist.html#concept_board` | 板块涨跌幅、领涨股、成交额、上涨家数 |
| **行业板块排行** | `http://quote.eastmoney.com/center/boardlist.html#industry_board` | 同上但按申万行业分类 |
| 板块成分股 | `http://quote.eastmoney.com/bk/<板块代码>.html` | 板块内所有股票列表 + 各自涨幅 |
| 龙虎榜 | `http://data.eastmoney.com/stock/lhb.html` | 哪些游资席位/机构在买什么 |
| 北向资金 | `http://data.eastmoney.com/hsgt/index.html` | 北向当日净流入、行业偏好 |

**抽取方式**：东财页面绝大多数是 SSR + 后续 JS 渲染表格。等待 DOM 稳定后用 accessibility tree 拿行表，比截图 OCR 更精确、更稳定。**遇到表格被懒加载（如下拉才出）时优先 scroll 触发渲染再抽，仍然不行才截图传 vision**。

### 同花顺行情（交叉验证用） — https://stockpage.10jqka.com.cn/<代码>/

仅作为东财的**交叉验证**。两边数字差超过 5% → 标记"数据冲突"，按巨潮 / 交易所原文为准。

⚠️ **不允许调用同花顺 i问财（iwencai.com）。** 它是 LLM 中介。

### 同花顺涨停板复盘（情绪温度交叉验证）
- URL：`http://q.10jqka.com.cn/zt/`（涨停板池）/ `http://q.10jqka.com.cn/dt/`（跌停板池）
- 用途：与东财涨停板池对比，差异 > 3 只 → 标记"数据冲突"

## 第三档：券商研报（仅取数字事实）

### 东方财富研报频道 — http://data.eastmoney.com/report/
- **只取数字事实**：研报里的"我们预计 2026 年净利润 X 亿元"——这种**预测数字不可直接用**，但研报里引用的"公司近一期实际数字"或"行业数据（如某新能源材料价格）"可以采用
- **永远不写"某某券商给出买入/增持/目标价 X"**作为推荐理由

## 拒绝来源（违反即等于编造）

- ❌ **同花顺 i问财、AI 助手、AI 选股**——任何 LLM 中介
- ❌ 股吧、雪球热帖、东方财富股吧
- ❌ 公众号、知乎、小红书
- ❌ 短视频截图、群聊截图、朋友推荐
- ❌ 未署名"市场传闻""听说"
- ❌ 隔夜消息但当日大盘已反映过的旧文章

## 数据完整性检查

每只候选必须能交叉验证：

1. **代码归属**：前缀符合主板（600/601/603/605/000/001/002/003），不在排除清单
2. **价格一致**：东财价 vs 同花顺价相差 < 0.5%（盘后两边数据应已同步）
3. **财报口径**：扣非净利润 vs 经营现金流偏离 > 30% → 警惕"利润高但现金没回来"（应收账款堆积）
4. **公告与行情**：今天大涨但近 30 日无公告且板块不在主线 → 标记"无逻辑放量"，进回避池
5. **板块归属**：东财板块 vs 同花顺板块归类如有差异，以东财为准但报告中说明

## agent-browser 操作约定

- **每次 session 开始先跑** `agent-browser skills get core` 拿最新指令——不要从这份文档复制 navigate/click 命令
- **优先 accessibility tree 抽原文**；仅当 DOM 拿不到时用截图 + vision，**截图读完即弃，不归档**
- **同一夜复用同一个 browser session**（节省登录态/性能）
- **每个数据点记录**：来源 URL + 抓取时间（不是"今晚最新"，是"22:14 抓"）
- 抓取失败 / 网站异常 → 写到报告"数据缺口"段，**不要硬猜或补编**
- **不要使用任何会让 LLM 改写真值的中间层**——只读原始页面
